# Re:Fail 기능 구현 현황

## 핵심 사용자 기능
- 게시글 작성·조회·수정·삭제
- 마크다운 작성, 미리보기, 상세 렌더링
- 다음 시도 선택 기록
- 후속 기록 작성·조회·수정·삭제
- 후속 상태: 다시 시도 중, 잠시 멈춤, 조금씩 나아지는 중, 극복함
- 내 기록 보관함: 익명·닉네임 게시글과 후속 기록 관리
- 브라우저 자동 임시 저장 및 복원
- 실패 회고 작성 가이드

## 탐색 기능
- 최신순·공감순 정렬
- 카테고리 필터
- 실패 크기 필터
- 제목·본문·감정 태그 검색
- 카테고리 이름·slug 검색
- 페이지 이동

## 공감과 안전
- 공감 유형별 개수 표시
- 로그인 사용자의 공감 상태 복원
- 재시도 의사에 맞는 공감 우선 노출
- 본인 게시글 공감 금지
- 댓글 기능 제외
- 게시글 신고

## 관리자 기능
- 신고 상태별 목록과 신고 사유 조회
- 게시글 숨김·복구
- 카테고리 생성·수정·비활성화 API
- 전체 게시글, 후속 기록률, 대기 신고, 재시도·중단·극복 지표

- 관리자: `admin@refail.local` / `password123`

## 시연 데이터 재생성
백엔드와 MySQL 실행 후 다음 명령을 사용한다. 이미 데이터가 있으면 중복 생성하지 않는다.

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\seed-demo-data.ps1
```
