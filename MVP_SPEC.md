# MVP 명세서

## 1. 목표

이 MVP는 포트폴리오용 프로젝트를 위한 것이다.

목표는 실제 대규모 커뮤니티 운영이 아니라 다음을 보여주는 것이다.
- 문제 정의가 명확한 제품 기획
- 이유가 분명한 기능 결정
- 현실적인 백엔드 설계
- 실제로 동작하는 사용자 흐름

## 2. MVP 제품 정의

사용자가 익명 또는 닉네임으로 실패를 공유하고, 공감 중심 리액션을 받고, 나중에 업데이트나 성공 후속 기록을 같은 글에 이어 붙일 수 있는 텍스트 중심 커뮤니티 사이트를 만든다.

## 4. 주요 사용자 흐름

### 흐름 1. 로그인 없이 읽기
1. 사용자가 홈에 들어온다.
2. 메인 메시지와 최근 실패 글을 본다.
3. 글 상세 페이지를 연다.
4. 실패 이야기, 공감 수, 댓글, 후속 기록 타임라인을 읽는다.
5. 반응이나 댓글을 남기려면 가입을 유도받는다.

### 흐름 2. 회원가입 후 실패 글 작성
1. 사용자가 회원가입한다.
2. 닉네임을 설정한다.
3. 글쓰기 화면으로 이동한다.
4. 익명 또는 닉네임 공개를 선택한다.
5. 제목, 내용, 카테고리, 실패 크기, 감정 태그, 위로/조언 선호를 입력한다.
6. 글을 등록한다.
7. 게시글이 홈 피드와 상세 화면에 노출된다.

### 흐름 3. 공감 남기기
1. 로그인한 사용자가 글 상세 화면을 연다.
2. 공감 리액션 하나를 선택한다.
3. 리액션 결과가 즉시 반영된다.

### 흐름 4. 후속 업데이트 추가
1. 원 작성자가 자신의 글을 연다.
2. 업데이트 추가를 선택한다.
3. 현재 상태를 선택한다.
- 아직 실패 중
- 다시 시도 중
- 나아지는 중
- 성공함
4. 무엇이 달라졌는지 작성한다.
5. 업데이트가 원글 타임라인에 연결된다.

### 흐름 5. 유해 콘텐츠 신고
1. 로그인한 사용자가 글의 신고 버튼을 누른다.
2. 신고 사유를 선택한다.
3. 신고가 저장된다.
4. 관리자가 숨김 또는 유지 여부를 결정한다.

## 5. 핵심 화면

### 1. 홈
목적
서비스 메시지를 보여주고 실패 글을 부담 없이 읽게 만든다.

필수 요소
- 메인 타이틀
- 짧은 소개 문구
- 글쓰기 CTA
- 최신순 / 인기순 탭
- 카테고리 필터
- 실패 글 카드 목록

### 2. 로그인 / 회원가입
목적
작성과 댓글을 위한 최소한의 진입 장벽만 둔다.

필수 요소
- 이메일과 비밀번호 또는 단순 소셜 로그인
- 닉네임 설정
- 익명 작성과 닉네임 작성에 대한 짧은 안내

### 3. 실패 글 작성
목적
실패 글을 부담 없이 쉽게 쓰게 만든다.

필수 요소
- 제목
- 내용
- 카테고리
- 실패 크기
- 감정 태그
- 익명 또는 닉네임 선택
- 위로 / 조언 선호 선택
- 등록 버튼

### 4. 게시글 상세
목적
실패 이야기와 그 이후 흐름을 함께 보여준다.

필수 요소
- 본문
- 작성 방식 표시
- 공감 리액션
- 신고 버튼
- 업데이트 타임라인
- 작성자용 업데이트 추가 버튼

### 5. 업데이트 작성
목적
실패 이야기가 이후에도 이어질 수 있게 한다.

필수 요소
- 연결된 원글 요약
- 현재 상태 선택
- 업데이트 내용 작성
- 등록 버튼

### 6. 관리자 모더레이션
목적
포트폴리오에서 운영 사고도 보여준다.

필수 요소
- 신고된 글 목록
- 숨김 / 복구 기능
- 계정 제한 기능

## 6. 세부 기능 규칙

### 계정과 정체성
- 글쓰기, 공감, 신고는 로그인 필요
- 계정당 닉네임은 하나
- 글마다 익명 또는 닉네임 공개를 선택할 수 있다
- 익명 글도 내부적으로는 계정 소유 글이다
- 공개 프로필이 생기더라도 익명 글은 노출하지 않는다

### 실패 글
- 반드시 하나의 카테고리를 가진다
- 반드시 하나의 작성 방식이 선택되어야 한다
- 반드시 위로/조언 선호를 선택한다
- 작성자는 수정과 삭제를 할 수 있다
- 성공은 독립 게시글로 작성할 수 없다

### 업데이트
- 업데이트는 하나의 원본 실패 글에 속한다
- 원 작성자만 업데이트를 작성할 수 있다
- 업데이트는 여러 개 작성 가능하다
- 성공은 업데이트 상태값으로 표현한다

### 공감 리액션
- 한 사용자는 한 글에 하나의 리액션만 선택 가능
- 리액션은 지지형 문구만 사용한다

추천 리액션
- 나도 그래
- 응원 보낼게요
- 공유해줘서 고마워요
- 다음 시도를 응원해요

### 댓글
댓글 기능은 MVP에서 제외한다.

제외 이유
- 실패 공유 서비스 초기에 가장 위험한 상호작용이 될 수 있다
- 운영 비용과 모더레이션 부담이 급격히 증가한다
- 현재 목표는 토론보다 기록과 공감에 있다

### 신고와 모더레이션
- 글은 신고 가능하다
- 신고 사유는 필수다
- 관리자는 콘텐츠를 숨길 수 있다
- 반복 악용 사용자는 제한할 수 있다
- 삭제나 숨김은 추적 가능하도록 소프트 삭제를 우선한다

## 7. 포트폴리오용 데모 범위

### 추천 데모 방식
실제로 동작하는 MVP와 시드 데이터를 함께 사용한다.

### 시드 데이터 구성
- 작은 일상 실패 글
- 감정적으로 무거운 실패 글
- 익명 글
- 닉네임 글
- 업데이트가 있는 글
- 성공까지 이어진 글 1개

### 데모에서 보여줘야 할 것
- 홈 피드 탐색
- 실패 글 작성
- 공감 남기기
- 업데이트 추가
- 신고된 게시글 모더레이션

### 데모에서 꼭 완벽하지 않아도 되는 것
- 운영 환경 수준의 보안 강화
- 대규모 모더레이션 기능
- 전체 알림 시스템
- 고급 분석 도구

## 10. 완료 기준

다음이 가능하면 MVP 완성으로 본다.
- 사용자가 회원가입하고 닉네임을 만들 수 있다
- 사용자가 익명 또는 닉네임으로 실패 글을 작성할 수 있다
- 사용자가 최신순 또는 인기순으로 글을 볼 수 있다
- 사용자가 카테고리별로 글을 필터링할 수 있다
- 사용자가 공감 리액션을 남길 수 있다
- 원 작성자가 업데이트를 추가할 수 있다
- 성공이 실패 타임라인의 일부로 보인다
- 사용자가 유해 콘텐츠를 신고할 수 있다
- 관리자가 신고된 콘텐츠를 숨길 수 있다

## 11. 명확한 비목표

이 MVP가 증명하려는 것이 아닌 것
- 바이럴 성장
- 대규모 운영 시스템
- 고급 소셜 네트워크 기능
- 미디어 플랫폼 기능
- 실시간 대규모 운영 경험

이 MVP가 증명하려는 것
- 생각이 있는 제품 설계
- 명확한 스코프 관리
- 일관된 백엔드 모델링
- 끝까지 연결되는 구현 능력
